about // profile.md

I'm an Associate Cloud & DevOps Engineer at Infanion Software Solutions, where I design, provision, and operate cloud infrastructure that real production traffic runs on — not just proof-of-concept environments. My work sits at the intersection of infrastructure automation, reliability, and cost-aware architecture.

Day to day, that means writing Terraform to keep environments reproducible and version-controlled, wiring up load balancing and auto-scaling so applications stay responsive under changing load, and building observability with Prometheus, Grafana, and CloudWatch so issues get caught before they become outages.

I work comfortably across both AWS and Azure, moving between the two depending on what a project needs — whether that's EC2 and RDS on one stack or App Service and Azure Front Door on the other. I hold the AWS Certified Solutions Architect – Associate certification and I'm always looking to deepen that cloud and automation toolkit further.

▶ era.be
shipped
Designed and managed the AWS infrastructure end-to-end using Terraform for reproducible, version-controlled provisioning. EC2 instances sit behind an Application Load Balancer for traffic distribution, with Auto Scaling configured to add or remove instances based on load, keeping the app responsive under varying traffic without over-provisioning. AWS WAF is attached at the edge to filter malicious requests before they reach the application layer. Data is served through RDS for durable relational storage, with ElastiCache in front of it to cut down repeated database load and speed up frequent reads. CloudWatch ties it all together with metrics, logs, and alarms across the stack, so issues surface before they become outages.
EC2 Auto Scaling Load Balancer WAF RDS ElastiCache CloudWatch Docker Terraform
▶ Bleckmann ENPS
shipped
Handled the AWS infrastructure for Bleckmann's ENPS platform, built around EC2 instances running behind a Load Balancer to distribute incoming traffic evenly and keep the app available if any single instance has issues. Auto Scaling adjusts instance count automatically as demand shifts, so the platform stays performant during peak load without paying for idle capacity the rest of the time. CloudFront sits in front as the CDN layer, caching static content at edge locations closer to users for faster load times and reduced load on the origin servers. Prometheus and Grafana handle metrics collection and visualization, giving continuous visibility into system health and surfacing anomalies early.
EC2 Load Balancer Auto Scaling CloudFront Prometheus Grafana
▶ DailyWorkManager
shipped
Built and managed the Azure infrastructure for DailyWorkManager using a split hosting approach — Azure App Service runs the backend application logic, while the frontend is served separately through Azure Static Web Apps for fast, CDN-backed delivery. Azure Front Door sits at the entry point, providing global routing, load balancing, and an additional layer of edge acceleration and security in front of both services. Azure Key Vault centralizes secrets, connection strings, and credentials so sensitive configuration never lives in application code. Data persistence runs on Azure Database for PostgreSQL, with Azure Cache for Redis layered in front to reduce database load and speed up frequently accessed data.
Azure App Service Azure Static Web App Azure Front Door Azure Key Vault Azure Database for PostgreSQL Azure Cache for Redis
